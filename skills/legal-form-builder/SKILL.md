# Legal Form Builder — สร้างเอกสารสัญญาแบบฟอร์มกรอก

สร้างเอกสารสัญญาและหนังสือทางกฎหมายภาษาไทยในรูปแบบ **"ฟอร์มกรอก"**
ที่ใช้ตาราง 2 column ให้กรอกด้วยปากกาได้สะดวก เห็นข้อมูลชัด เป็นมืออาชีพ

---

## หลักการสำคัญ — ทำไมต้อง "ฟอร์มกรอก"

**ปัญหาของ format แบบ paragraph ที่มีช่องเส้น:**
- กรอกยาก ต้องเล็งให้ตรงเส้น
- ดูยุ่ง อ่านลำบาก
- print แล้วถ้ากรอกผิดต้องแก้ทั้งย่อหน้า

**ข้อดีของ "ฟอร์มกรอก" แบบตาราง 2 column:**
- กรอกง่าย — แต่ละช่องชัดเจน เป็นระเบียบ
- ดูเป็นมืออาชีพ — เห็นเป็นแถว ๆ เหมือนเอกสารราชการ
- print แล้วเขียนด้วยปากกาได้สะดวก ไม่หลุดเส้น
- ตรวจสอบง่าย — ดูปุ๊บรู้ว่าข้อมูลส่วนไหนอยู่ตรงไหน

---

## Input ที่ต้องได้ก่อนทำงาน

1. **ประเภทเอกสาร** (NDA, สัญญาจ้าง, หนังสือยินยอม ฯลฯ)
2. **ผู้เซ็นคือใคร** (พนักงาน, freelance, vendor, ลูกค้า)
3. **บริบท / วัตถุประสงค์** ของเอกสาร
4. **เงื่อนไขเฉพาะ** (เช่น ค่าปรับ, ระยะเวลาผูกพัน, มาตราที่ต้องอ้าง)

ถ้าไม่มี → ถามก่อน อย่าเดา

---

## ขั้นตอน

### STEP 1 — ค้นข้อมูลกฎหมายก่อนเสมอ

ก่อนร่างเอกสาร ต้องค้น `project_knowledge_search` หา:
- มาตราในกฎหมายที่เกี่ยวข้อง (พ.ร.บ.คุ้มครองแรงงาน, ป.พ.พ., พ.ร.บ.เฉพาะเรื่อง)
- ฎีกาที่เกี่ยวข้อง (ถ้ามีใน wiki)
- ข้อหารือกรมแรงงาน (ถ้าเกี่ยวกับแรงงาน)

**ห้ามมั่วเลขมาตราหรือเลขฎีกาเด็ดขาด** — ถ้าไม่แน่ใจให้บอกตามตรง

### STEP 2 — วิเคราะห์ความเสี่ยงทางกฎหมาย 3 จุด

ก่อนร่าง ต้องคิดให้ครบ 3 ข้อ:

1. **มาตรา 14/1 พ.ร.บ.คุ้มครองแรงงาน** — เอกสารที่ทำให้นายจ้างได้เปรียบ "เกินสมควร"
   ศาลมีอำนาจปรับลด → ต้องมีข้อยกเว้น มีเหตุผลรองรับ
2. **มาตรา 10 (เรื่องหลักประกัน)** — ห้ามเรียกหลักประกันจากลูกจ้าง
   เว้นแต่งานเกี่ยวกับการเงิน → ค่าปรับต้องเป็น "Liquidated Damages" ไม่ใช่หลักประกัน
3. **ป.พ.พ. มาตรา 383** — เบี้ยปรับสูงเกินส่วน ศาลปรับลดได้
   → ต้องมี logic รองรับว่าทำไมจึงตัวเลขนั้น

### STEP 3 — โครงสร้างเอกสาร (มาตรฐาน)

```
1. Header (โลโก้/ชื่อบริษัท + ที่อยู่)
2. Title block (ชื่อเอกสารใหญ่ + คำขยายภาษาอังกฤษ)
3. ตาราง "ทำที่/วันที่"          ← ฟอร์มกรอก
4. ตารางคู่สัญญาฝ่ายที่ 1         ← ฟอร์มกรอก
5. คำว่า "กับ" (กลางหน้า)
6. ตารางคู่สัญญาฝ่ายที่ 2         ← ฟอร์มกรอก
7. ตารางข้อมูลโครงการ/ขอบเขต      ← ฟอร์มกรอก (ถ้ามี)
8. คำปรารภ (preamble)
9. ข้อ 1, 2, 3, ... (เนื้อหาสัญญา)
10. ตารางค่าเสียหาย/บทลงโทษ      ← ตารางข้อมูล
11. ข้อกำหนดทั่วไป
12. ส่วนลายเซ็น
13. คำรับรองการอ่าน (ถ้าจำเป็น)
```

## Style มาตรฐาน (บังคับใช้ทุกครั้ง)

### Color Palette

```javascript
const NEAR_BLACK = "1A1A1A";   // ตัวอักษรหลัก
const DARK_GRAY  = "2D2D2D";   // (ไม่ใช้กับ table header แล้ว)
const MID_GRAY   = "595959";   // text รอง
const LINE_GRAY  = "D4D4D4";   // border
const SOFT_BG    = "F8F8F8";   // label cell ซ้ายของตารางฟอร์ม
const HEADER_BG  = "C8C8C8";   // header ของ section ในตาราง — เทาอ่อน
```

**กฎสำคัญ — สีของ header:**
- ห้ามใช้ DARK_GRAY (สีเข้ม + ตัวขาว) สำหรับ header ตาราง
- ใช้ HEADER_BG (C8C8C8) + ตัวอักษร NEAR_BLACK เสมอ
- เพราะดูสุภาพ อ่านง่าย ไม่ดูแข็งกระด้าง
- HEADER_BG ต้องเข้มกว่า SOFT_BG ชัดเจน (ห่างกันอย่างน้อย 30 step ของ RGB)
  ไม่งั้นแยก header กับ row label ไม่ออก

### Font

- **Niramit** ทั้งเอกสาร
- body: 10pt (size: 20)
- header table cell: 11pt (size: 22) bold
- title ใหญ่: 22pt (size: 44) bold
- subtitle: 14pt (size: 28)

### ตารางฟอร์มกรอก — กฎเหล็ก

**โครงสร้างทุกตารางฟอร์ม:**
```
column 1 (label):  width 2400 DXA, shading SOFT_BG, bold
column 2 (ช่อง):    width 6720 DXA, ใส่ "____________________________________________________"
รวม: 9120 DXA (= content width ของ US Letter ที่ margin 1080 ทั้งสองข้าง)
```

**Section header row (สำหรับเริ่มกลุ่มใหม่):**
- columnSpan: 2 (กินทั้งความกว้าง)
- shading: HEADER_BG (C8C8C8)
- color: NEAR_BLACK
- bold: true
- size: 22 (11pt)

**ห้ามทำ:**
- ❌ ใส่ข้อความตัวอย่างไว้ในช่องกรอก เช่น "บริษัท ABC จำกัด"
- ❌ ใช้ paragraph + ".................." แทนตาราง
- ❌ ใช้ DARK_GRAY กับตัวอักษรขาวเป็น header
- ❌ ผสมรูปแบบ — ทุกส่วนที่กรอกต้องเป็นตาราง

**ต้องทำ:**
- ✅ ใช้ "____________________________________________________" (52 underscore) เป็นช่องกรอก
- ✅ แยกแต่ละ field เป็น 1 row ในตาราง
- ✅ field ที่ต้องกรอกหลายส่วนในบรรทัดเดียว ใช้ตามนี้: "วันที่ ____________ เดือน ____________________ พ.ศ. ____________"

### Page Setup

```javascript
size: { width: 12240, height: 15840 },         // US Letter
margin: { top: 1080, right: 1560, bottom: 1080, left: 1560 }
// content width = 12240 - 1560*2 = 9120 DXA
```

### Header & Footer

**Header:**
- ชื่อบริษัท bold (ใส่ placeholder `[ชื่อบริษัท]` ถ้าไม่มี)
- ที่อยู่ MID_GRAY ใต้ลงไป
- เส้นใต้สีดำ NEAR_BLACK

**Footer:**
- เส้นเทาด้านบน
- ซ้าย: ชื่อเอกสาร MID_GRAY size 16 (8pt)
- ขวา: "หน้า X / Y" (PageNumber.CURRENT / PageNumber.TOTAL_PAGES)

## ประเภทเอกสารที่รองรับ

| ประเภท | ฟอร์มกรอกที่ต้องมี |
|---|---|
| NDA / หนังสือรักษาความลับ | คู่สัญญา 2 ฝ่าย + ข้อมูลโครงการ |
| สัญญาจ้างพนักงาน | นายจ้าง + พนักงาน + ตำแหน่ง/เงินเดือน/ระยะเวลา |
| หนังสือยินยอม PDPA | ผู้ให้ความยินยอม + วัตถุประสงค์ |
| หนังสือมอบอำนาจ | ผู้มอบอำนาจ + ผู้รับมอบอำนาจ + ขอบเขตอำนาจ |
| สัญญาเช่า | ผู้ให้เช่า + ผู้เช่า + รายละเอียดทรัพย์สิน |
| สัญญาซื้อขาย | ผู้ขาย + ผู้ซื้อ + รายละเอียดสินค้า/ราคา |
| สัญญาจ้างทำของ | ผู้ว่าจ้าง + ผู้รับจ้าง + งาน/ค่าจ้าง/กำหนดส่ง |
| MOU | คู่สัญญา 2+ ฝ่าย + ขอบเขตความร่วมมือ |
| หนังสือค้ำประกัน | ผู้ค้ำประกัน + ลูกหนี้ + เจ้าหนี้ + ภาระผูกพัน |
| สัญญาประนีประนอม | คู่กรณี + ข้อพิพาท + เงื่อนไขประนีประนอม |

## Validation ก่อนส่ง

ทุกครั้งหลัง generate ไฟล์ ต้อง:

1. รัน `python3 /mnt/skills/public/docx/scripts/office/validate.py [file.docx]`
2. ถ้ามีเวลา/สงสัย → render เป็น PDF ตรวจสายตา:
   ```bash
   python3 /mnt/skills/public/docx/scripts/office/soffice.py --headless --convert-to pdf [file.docx]
   pdftoppm -jpeg -r 100 [file.pdf] page
   ```
   แล้ว view page-1.jpg เช็คว่าตารางฟอร์มกรอกถูกต้องไหม
3. ตรวจ pixel ของ header ว่าได้ HEADER_BG (C8C8C8) จริงไม่ใช่ default

## Rules

- **ค้นกฎหมายก่อนเสมอ** — ใช้ `project_knowledge_search` หาเลขมาตรา ห้ามมั่ว
- **ใช้ HEADER_BG = C8C8C8 เสมอ** ไม่ใช่ DARK_GRAY (พี่ปุ้ย confirm แล้วว่าสีนี้ดูดีที่สุด)
- **ทุกส่วนที่ต้องกรอก = ตารางฟอร์ม** ไม่ใช่ paragraph
- **ห้ามใส่ข้อความตัวอย่างในช่องกรอก** — ปล่อยให้พี่กรอกเอง
- **ตัวเลขเงิน/วันเวลา ในข้อกฎหมาย** ตามที่พี่ระบุเท่านั้น ห้ามเปลี่ยนเอง
- **ปิดความเสี่ยง 3 จุด** (ม.14/1, ม.10, ป.พ.พ.383) เสมอเมื่อทำเอกสารแรงงาน
- **validate ก่อนส่งทุกครั้ง** ห้าม present ไฟล์ที่ไม่ผ่าน validation
- **ใส่คำรับรองการอ่าน** ท้ายเอกสารทุกฉบับที่มีบทลงโทษ — ป้องกันอ้างภายหลัง

## Next Action หลังสร้างเสร็จ

หลัง present ไฟล์แล้ว ต้องบอกพี่ปุ้ย:

1. **จุดที่ต้องกรอก** มีกี่ field
2. **มาตรากฎหมายที่อ้างอิง** ในเอกสารนี้
3. **ความเสี่ยง 3 จุด** ที่ปิดไว้แล้ว (ถ้าเกี่ยวกับแรงงาน)
4. **คำแนะนำขั้นตอนการลงนาม** — ใครต้องเซ็นก่อน, มีพยานกี่คน, เก็บกี่ฉบับ
